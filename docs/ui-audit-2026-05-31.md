# FlowTO — UI Audit (2026-05-31)

Playwright-driven walkthrough of the running app (`localhost:5173` → API `:8000`),
auditing both **Simulate** and **Edit** modes, light + dark themes, the Copilot
panel, and three viewport widths (1440 / 768 / 390). Findings are grouped by
severity with a concrete fix and source pointer for each. Nothing here is fixed
yet — this is a punch-list to work through later.

Screenshots referenced below live in [`ui-audit-assets/`](./ui-audit-assets/).

---

## P0 — Breaks the experience

### 1. No responsive layout at all — app is unusable below ~1280px
The entire shell is a fixed multi-column desktop grid with **zero media queries**
(`grep -r "@media" frontend/src` → no matches). Consequences:

- **768px (tablet, `audit-12-tablet-768.png`):** the map collapses to a thin
  vertical sliver; the right Inspector/Copilot dock overlaps the map column;
  panels don't reflow.
- **390px (phone, `audit-11-mobile-390.png`):** the map is pushed entirely
  off-screen, the right dock is clipped past the viewport edge, and the top tab
  bar is cut off. Effectively a broken page.

**Fix:** decide a real strategy — at minimum a `min-width` guard with a "best
viewed on desktop" notice, ideally a responsive mode that collapses the side
docks into drawers/tabs below a breakpoint and lets the map take the full width.
The dock-toggle buttons (see #5) already hide panels individually, so the
plumbing for a collapsed layout partly exists.
**Where:** the app shell CSS / grid (no `@media` anywhere in `frontend/src`).

## P1 — Visible rough edges

### 2. Copilot surfaces raw HTTP errors to the user
When `/api/copilot/plan` fails, the chat bubble reads literally
**"Copilot unavailable: POST /copilot/plan → 500"** (`audit-09-copilot-error.png`).
That's an internal error string leaking method/path/status into a user-facing
surface. (In this local session the 500 is just the Nemotron LLM backend being
unreachable over Tailscale — but the *handling* is the UI bug, independent of
why it failed.)

**Fix:** map failures to a friendly message ("Copilot is offline right now —
the model backend isn't reachable. Retry?") with a retry affordance; log the raw
detail to console only.
**Where:** `frontend/src/state/appStore.ts:628` — `text: \`Copilot unavailable: ${msg}\``.

### 3. Dock-toggle buttons use cryptic block glyphs
The three header buttons that show/hide the left/bottom/right docks render as
Unicode blocks `▘ ▂ ▝` (`audit-03-topbar.png`). They *do* have `title` tooltips
("Left dock" etc.), so they're not inaccessible — but visually they read as
noise and the active/inactive state is hard to parse.

**Fix:** replace with real panel-layout icons (the codebase already has an `Icon`
set — `Icon.moon` is used right beside them) and give a clearer on/off treatment.
**Where:** `frontend/src/components/TopBar.tsx:49–59`.

### 4. Missing favicon → 404 on every load
`index.html` declares no `<link rel="icon">`, so the browser requests
`/favicon.ico` and gets a 404 (console error on first paint).

**Fix:** add a favicon (even an inline SVG `data:` link) to
`frontend/index.html`. Quick win, removes the only "real" console error.

## P2 — Polish / refinement

### 5. Empty panels waste vertical space
Several docks are mostly blank in their default/empty state:
- **Saved simulations** (left, Simulate mode) — name field + Save, then a huge
  empty column (`audit-06-leftpanel.png`).
- **Warnings** (right) — two cards, then large empty area (`audit-04-rightpanel.png`).
- **Copilot** — intro line, then a big gap before the suggestion chips, which sit
  pinned near the bottom (`audit-04-rightpanel.png`).

**Fix:** tighten empty states — center the empty-state copy, or pull the Copilot
suggestion chips up directly under the intro text instead of bottom-anchoring
them. Low effort, makes the panels feel intentional rather than unfinished.

### 6. Ambiguous "Network · 40%" label
The demand-level control in the timeline reads "Network · 40%" / "Network · 20%"
(`audit-05-timeline.png`) with no indication it's a demand-volume multiplier.

**Fix:** label it as what it is (e.g. "Demand · 40%" or a tooltip "share of full
egress demand"). Confirm against the engine semantic in `frontend/src/config.ts:44`.

### 7. Benign Mapbox console warning
`Cutoff is currently disabled on terrain` logs on every load. Harmless, but it's
console noise. Optional: set the terrain/cutoff option or suppress.
