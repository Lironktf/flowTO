- Title: `P0x — <phase title>` (stretch: `Sx — <title>`).
- Description = Goal (verbatim/condensed) + 2–3 line summary + spec link.
- Priority: core phases = **High** (Linear priority 2); stretch = **Low** (Linear priority 4).
- Estimate = sum of task day-estimates, rounded to the team's point scale (read the team's
  estimation setting first; if Fibonacci, days≈points; round 3.25→3, 4.5→5, 6.5→6 or 7, 7.0→7/8).
- Labels per issue: `phase`, (`core`|`stretch`), and ONE area label (`data`|`sim`|`frontend`|`ai`|`infra`).
- Status: Backlog/Todo. Unassigned.
- One sub-issue per task (T0x.y); estimate = that task's day estimate.
- Blocked-by relations exactly per the table below.
- Project ordering (critical path first): P00 → P01 → P02 → P03 → P04 → P06 → P07 → P12,
  then P05, P08, P09, P10, P11, then S1..S6.
